# DIF (DocIntelliFix) 部署指南

> 本文档用于指导在导师电脑或服务器上部署 DIF 项目。
> 
> 最后更新：2026 年 4 月

---

## 一、项目简介

**DIF (DocIntelliFix)** 是一款智能文档修复与增强系统，集成了图像超分辨率重建、OCR 文字识别、文本纠错等功能。

### 核心功能

| 功能 | 技术 | 说明 |
|------|------|------|
| 图像超分 | Swin2SR (PyTorch) | 4x 超分辨率，支持 realworld/classical/compressed 三种模型 |
| OCR 识别 | RapidOCR v5 (ONNX) | GPU 加速，支持中文检测与识别 |
| 文本纠错 | MacBERT (PyTorch) | 智能拼写纠错，保护格式和非中文字符 |
| 文本补全 | 规则引擎 + Qwen (可选) | 语义补全和拼写纠错 |
| 用户管理 | JWT + SQLite | 完整的用户认证和历史记录 |

### 技术架构

```
┌─────────────┐        ┌──────────────────┐        ┌──────────────┐
│  Vue 3 前端  │───────▶│  FastAPI 后端     │───────▶│  AI 模型     │
│  (Vite)     │        │  (Python 3.10+)  │        │  (GPU/CPU)   │
│  :5173      │◀───────│  :8000           │◀───────│              │
└─────────────┘        └──────────────────┘        └──────────────┘
```

## 二、环境要求

### 最低要求

| 组件 | 版本 | 备注 |
|------|------|------|
| 操作系统 | Windows 10/11 | 或 Linux/macOS |
| Python | 3.10+ | 后端运行环境 |
| Node.js | 18+ | 前端构建环境 |
| npm | 9+ | 前端依赖管理 |

### GPU 加速（可选但强烈推荐）

| 组件 | 版本 | 说明 |
|------|------|------|
| NVIDIA GPU | RTX 3060 或更高 | 显存 6GB+ |
| CUDA | 11.7+ | 与 PyTorch 版本匹配 |
| cuDNN | 8.x | 与 CUDA 版本匹配 |

> ⚠️ **如果没有 GPU**：系统仍可在 CPU 上运行，但 AI 推理速度会慢很多（约 3-5 倍）。

## 三、部署步骤

### 第一步：克隆项目

```bash
git clone <repository-url>
cd DIF
```

### 第二步：安装 Python 环境

#### 1. 创建虚拟环境（推荐）

```bash
# Windows PowerShell
python -m venv venv
.\venv\Scripts\Activate.ps1

# 或者使用 conda
conda create -n dif python=3.10
conda activate dif
```

#### 2. 安装后端依赖

```bash
cd backend
pip install -r requirements.txt
```

**依赖安装注意事项**：

- 如果遇到 `torch` 安装缓慢，可使用清华镜像源：
  ```bash
  pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
  ```

- 如果需要使用 GPU 加速，请安装 CUDA 版本的 PyTorch：
  ```bash
  pip uninstall torch torchvision
  pip install torch==1.13.1+cu117 torchvision==0.14.1+cu117 -f https://download.pytorch.org/whl/torch_stable.html
  ```

#### 3. 下载预训练模型

系统需要以下预训练模型文件（不纳入 Git 仓库，需手动下载）：

**Swin2SR 模型**（放在 `backend/weights/swin2sr/` 目录）：

| 模型 | 用途 | 下载链接 |
|------|------|----------|
| Swin2SR_RealworldSR_X4_64.pth | 真实世界图像超分（推荐） | [GitHub](https://github.com/mv-lab/swin2sr/releases) |
| Swin2SR_CompressedSR_X4_48.pth | 压缩图像恢复 | [GitHub](https://github.com/mv-lab/swin2sr/releases) |
| Swin2SR_ClassicalSR_X4_64.pth | 经典图像超分 | [GitHub](https://github.com/mv-lab/swin2sr/releases) |

**MacBERT 模型**（首次运行时自动从 HuggingFace 下载）：

- 模型名：`shibing624/macbert4csc-base-chinese`
- 无需手动下载，程序会自动下载

**RapidOCR 模型**（首次运行时自动下载 ONNX 模型）：

- 无需手动下载，程序会自动下载

#### 4. 配置环境变量

```bash
cd backend
copy .env.example .env
# 或使用文本编辑器打开 .env 文件
```

编辑 `.env` 文件，修改以下关键配置：

```env
# JWT 密钥（必须修改！）
JWT_SECRET=your-new-random-secret-key-here

# Token 过期时间（0 = 永不过期）
JWT_EXPIRE_MINUTES=0
```

生成随机 JWT_SECRET 的方法：
```bash
python -c "import secrets; print(secrets.token_urlsafe(32))"
```

#### 5. 创建管理员账户

```bash
cd backend
python create_admin_user.py
```

### 第三步：安装前端环境

```bash
cd frontend
npm install
```

**如果遇到 npm 安装缓慢**：

```bash
# 使用淘宝镜像源
npm config set registry https://registry.npmmirror.com
npm install
```

### 第四步：启动服务

#### 方式一：使用启动脚本（推荐）

```bash
# 在项目根目录下运行
.\start.bat
```

这会同时启动后端和前端两个终端窗口。

#### 方式二：手动启动

**终端 1 - 启动后端**：
```bash
cd backend
python -m uvicorn app.main:app --host 0.0.0.0 --port 8000
```

**终端 2 - 启动前端**：
```bash
cd frontend
npm run dev
```

### 第五步：验证部署

1. 打开浏览器访问：http://localhost:5173
2. 使用管理员账户登录
3. 上传一张测试图片，验证各项功能

## 六、常见问题

### 1. 后端启动报错 `ImportError: No module named xxx`

**原因**：Python 依赖未正确安装。

**解决**：
```bash
cd backend
pip install -r requirements.txt
```

### 2. 前端无法连接后端

**原因**：前后端地址不匹配或跨域问题。

**解决**：
- 确认后端在 8000 端口运行
- 确认前端在 5173 端口运行
- 前端已配置 Vite 代理，确保 `vite.config.ts` 中的代理配置正确

### 3. GPU 不可用 / CUDA 报错

**原因**：CUDA 版本与 PyTorch 不匹配。

**解决**：
```bash
# 查看 CUDA 版本
nvcc --version

# 安装匹配的 PyTorch（以 CUDA 11.7 为例）
pip install torch==1.13.1+cu117 torchvision==0.14.1+cu117 -f https://download.pytorch.org/whl/torch_stable.html
```

### 4. Swin2SR 模型未加载

**原因**：模型文件不存在或路径配置错误。

**解决**：
- 检查 `backend/weights/swin2sr/` 目录下是否有 `.pth` 文件
- 确认模型文件名与代码中的配置一致

### 5. 首次启动加载很慢

**原因**：模型首次下载和初始化需要时间。

**解决**：这是正常现象，首次启动可能需要 2-5 分钟。之后启动会快很多。

### 6. 端口被占用

**原因**：8000 或 5173 端口已被其他程序使用。

**解决**：
```bash
# 查看端口占用
netstat -ano | findstr 8000
taskkill /PID <进程ID> /F

# 或修改配置使用其他端口
```

### 7. npm install 失败

**原因**：网络问题或 Node.js 版本不兼容。

**解决**：
```bash
# 切换淘宝镜像
npm config set registry https://registry.npmmirror.com
npm install

# 确保 Node.js >= 18
node --version
```
